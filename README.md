# DivLauncher

## Описание проекта
Пользовательский лаунчер для Minecraft, разработанный на базе Electron и React. Приложение предоставляет интерфейс для выбора серверных сборок, автоматического скачивания обновлений файлов игры и управления процессом запуска клиента.

## Применяемые технологии
* **Среда выполнения:** Node.js, Electron
* **Интерфейс:** React, Framer Motion
* **Работа с сетью и файловой системой:** Axios, Adm-Zip, fs
* **Автоматические обновления:** electron-updater

## Требования к окружению
* Node.js (рекомендуется версия 16.x или выше)
* npm или yarn

## Установка и локальная разработка

1. Клонируйте репозиторий:
   ```bash
   git clone [https://ru.wikipedia.org/wiki/%D0%A0%D0%B5%D0%BF%D0%BE%D0%B7%D0%B8%D1%82%D0%BE%D1%80%D0%B8%D0%B9](https://ru.wikipedia.org/wiki/%D0%A0%D0%B5%D0%BF%D0%BE%D0%B7%D0%B8%D1%82%D0%BE%D1%80%D0%B8%D0%B9)
   ```

2. Перейдите в директорию проекта:
   ```bash
   cd DivLauncher
   ```

3. Установите зависимости:
   ```bash
   npm install
   ```

4. Запустите приложение в режиме разработки:
   ```bash
   npm start
   ```

## Сборка приложения

Для создания исполняемого файла (.exe) и генерации файлов манифеста обновлений выполните команду:
```bash
npm run build
```
Результаты сборки будут помещены в директорию `dist/`. Файл `latest.yml` и `.exe` инсталлятор предназначены для загрузки на удаленный сервер (Cloudflare R2) для обеспечения работы системы автообновлений лаунчера.

## Структура конфигурации сборок (modpacks.json)

Файл `modpacks.json`, размещенный на удаленном сервере, управляет доступными сборками. Лаунчер загружает этот файл при запуске для актуализации данных. 

Обязательные поля для сборок, использующих ZIP-архивы:
* `id`: Уникальный идентификатор сборки.
* `packVersion`: Текущая версия сборки (используется для сверки с локально установленной версией).
* `packZipUrl`: Прямая ссылка на ZIP-архив с модами и конфигурациями.
* `useZip`: Логический флаг (`true`), указывающий на необходимость скачивания и распаковки архива.
* `clientDir`: Имя локальной директории для установки файлов сборки (например, `.statech-client`).

## Механизм обновлений

В проекте реализованы две независимые системы обновлений:

1. **Обновление лаунчера (Electron Updater):** Проверяет наличие новой версии приложения через файл `latest.yml` на удаленном сервере.
2. **Обновление игровых сборок:** Сравнивает значение `packVersion` из `modpacks.json` с версией, сохраненной в `localStorage`. При несовпадении лаунчер загружает архив по ссылке `packZipUrl` и перезаписывает файлы в директории `clientDir`.

## Настройка хранилища Cloudflare R2

Для корректной работы системы автообновлений лаунчера и загрузки игровых сборок необходимо настроить бакет (bucket) в Cloudflare R2.

### 1. Создание бакета
1. Авторизуйтесь в панели управления Cloudflare и перейдите в раздел **R2 Object Storage**.
2. Нажмите **Create bucket** (Создать бакет).
3. Введите уникальное имя бакета (например, `divlauncher-data`) и выберите регион, ближайший к вашей целевой аудитории (или оставьте Automatic).
4. Нажмите **Create bucket**.

### 2. Настройка публичного доступа (Public Access)
Лаунчеру требуются прямые HTTP/HTTPS ссылки для загрузки файлов без использования токенов авторизации AWS S3.

1. Перейдите в настройки созданного бакета (вкладка **Settings**).
2. Прокрутите до раздела **Public Access**.
3. В подразделе **Custom Domains** нажмите **Connect Domain**.
4. Введите ваш подготовленный поддомен (например, `mc.diverlin.ru`) и подтвердите привязку. Cloudflare автоматически настроит DNS-записи и выпустит SSL-сертификат.
5. (Опционально) Можно также включить базовый доступ через домен `r2.dev`, нажав **Allow Access** в подразделе *R2.dev subdomain*, однако использование собственного домена предпочтительнее.

### 3. Настройка CORS (Обязательно для прогресс-бара)
Поскольку лаунчер использует `axios` для загрузки файлов и отслеживания прогресса, необходимо разрешить кросс-доменные запросы и чтение заголовка `Content-Length`. Без этого прогресс-бар загрузки не будет функционировать.

1. Во вкладке **Settings** вашего бакета прокрутите вниз до раздела **CORS rules**.
2. Нажмите **Add CORS policy** и переключитесь в режим редактирования JSON.
3. Вставьте следующую конфигурацию:

```json
[
  {
    "AllowedOrigins": [
      "*"
    ],
    "AllowedMethods": [
      "GET",
      "HEAD"
    ],
    "AllowedHeaders": [
      "*"
    ],
    "ExposeHeaders": [
      "Content-Length",
      "Content-Type"
    ],
    "MaxAgeSeconds": 3600
  }
]
```
4. Сохраните изменения.

### 4. Структура файлового хранилища
Для поддержания порядка рекомендуется соблюдать строгую иерархию папок в бакете. Все пути должны точно совпадать с теми, что указаны в `package.json` и `modpacks.json`.

Пример структуры:
```text
/
├── updates/                         # Обновления самого лаунчера (electron-updater)
│   ├── latest.yml                   # Генерируется при сборке (npm run build)
│   └── DivLauncher-Setup-1.0.1.exe  # Установочный файл
│
└── DivLauncher/                     # Данные лаунчера и игровых сборок
    ├── modpacks.json                # Главный конфигурационный файл
    ├── stalker/
    │   ├── assets/
    │   │   ├── bg.jpg
    │   │   ├── icon.png
    │   │   └── minecraft.png
    │   └── mods.json
    └── statech/
        ├── assets/
        │   ├── bg.jpg
        │   ├── icon.png
        │   └── banner.png
        └── statech_V101.zip         # ZIP-архив с модами (packZipUrl)
```

### 5. Рекомендации по обновлению файлов
Из-за агрессивного кэширования на стороне серверов Cloudflare (CDN) применяется следующее правило обновления игровых сборок:
* **Запрещено** заменять старый ZIP-архив новым файлом с точно таким же именем. Лаунчер может скачать старую версию из кэша.
* **Обязательно** изменяйте имя файла архива при каждом обновлении (например, `statech_V101.zip` -> `statech_V102.zip`).
* После загрузки нового архива обновите поле `packZipUrl` и инкрементируйте `packVersion` в файле `modpacks.json`. Кэширование файла `modpacks.json` отключено на стороне клиента программно.
