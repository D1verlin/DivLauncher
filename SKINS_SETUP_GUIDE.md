# Гайд по настройке системы скинов в DivLauncher

Эта инструкция объясняет, как настроить отображение скинов для пиратских и лицензионных игроков (Xbox Live) так, чтобы они корректно виделись всеми игроками на сервере.

---

## 1. Лицензионные игроки (Xbox Live)

Для игроков, которые авторизовались через **Xbox Live** в настройках лаунчера:

1. Скин загружается из аккаунта Microsoft.
2. При загрузке нового скина в лаунчере через кнопку **"Применить на Microsoft"** скин автоматически обновляется в аккаунте Mojang/Xbox.
3. **На лицензионном сервере (`online-mode=true`)** скин будет виден всем без дополнительных модов или плагинов.
4. **На пиратском сервере (`online-mode=false`)** скин будет виден другим только в том случае, если на сервере установлен плагин **SkinsRestorer** (или аналогичный), который восстанавливает скины для игроков, чей никнейм совпадает с лицензионным.

---

## 2. Пиратские игроки (Offline режим)

В пиратском режиме игрок загружает скин (файл `.png`) локально в лаунчере. Этот скин будет отображаться в 3D-просмотрщике лаунчера, но по умолчанию **другие игроки на сервере его не увидят** (они будут видеть дефолтного Стива или Алекс).

Чтобы пиратский скин был виден вам и другим игрокам на сервере, необходимо использовать клиентские моды или серверные плагины.

### Вариант А: Использование мода CustomSkinLoader (Рекомендуемый для сборок с модами)

Лаунчер автоматически сохраняет загруженный пиратский скин в папку `resourcepacks/skins/<никнейм>.png` внутри директории сборки. Мод **CustomSkinLoader** может подхватить этот скин и отображать его в игре.

**Инструкция для сборки:**
1. Установите мод [CustomSkinLoader](https://modrinth.com/mod/customskinloader) (доступен для Forge, Fabric, NeoForge).
2. Настройте `CustomSkinLoader/CustomSkinLoader.json` (в папке `.minecraft` или папке сборки), добавив локальную директорию `resourcepacks/skins/` как приоритетный источник загрузки скинов (`LocalSkin`).
3. Теперь при входе в игру мод прочитает сохраненный лаунчером локальный скин. *Чтобы этот скин видели другие игроки, у них тоже должен быть установлен этот мод и скин должен синхронизироваться через сторонний API (Ely.by, Cloaks+, и т.д.), либо вы должны использовать Вариант Б.*

### Вариант Б: Серверный плагин SkinsRestorer (Рекомендуемый для серверов)

Если вы владелец сервера, установите плагин **SkinsRestorer**, чтобы игроки могли менять скины прямо на сервере с помощью команд.

1. Установите плагин [SkinsRestorer](https://skinsrestorer.net/) на ваш сервер (Spigot / Paper / Purpur).
2. В пиратском режиме лаунчера игроку придется либо зайти под ником лицензионного игрока (тогда SkinsRestorer автоматически подтянет его скин), либо использовать команду на сервере: `/skin <url или ник>`.
3. Если у вас есть собственный сервер со своим лаунчером, вы можете настроить SkinsRestorer на использование вашего собственного API для получения скинов, куда лаунчер будет отправлять загруженные `.png` файлы (вместо локального сохранения).

---

## 3. Разработка собственного API для скинов (Продвинутый вариант)

Если вы хотите, чтобы пиратские скины, загруженные в лаунчере, автоматически были видны всем игрокам на вашем сервере **без ввода команд**, вам потребуется собственный сервер скинов (Skin Server).

**Архитектура:**
1. **Лаунчер**: При загрузке скина пиратом отправляет POST-запрос с `.png` файлом на ваш веб-сервер.
2. **Веб-сервер**: Сохраняет скин (например, `https://ваша-база.com/skins/<ник>.png`).
3. **Игровой Сервер / Клиент**: 
   - Настройте **CustomSkinLoader** в клиенте так, чтобы он в первую очередь проверял `https://ваша-база.com/skins/{USERNAME}.png`.
   - ИЛИ настройте **SkinsRestorer** / **CustomPayload** плагин на сервере для получения скинов с вашего веб-сервера.

*Текущая реализация лаунчера (сохранение в `resourcepacks/skins/<username>.png`) служит отличной базой для локального тестирования и работы через CustomSkinLoader в синглплеере/локальной сети.*
